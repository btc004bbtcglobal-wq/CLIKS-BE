# Work Done Today

## Development Updates

### 1. Security & Error Handling Enhancements
- **Global Error Handler:** Replaced raw errors with a structured, centralized error handler (`middleware/errorHandler.js`).
- **Zod Validation:** Discarded legacy `express-validator` logic in favor of robust schema validations with Zod (`middleware/zodValidate.js`).
- **Input Sanitization:** Intercepts and recursively trims all strings in `req.body`, `req.query`, and `req.params` (`middleware/sanitizer.js`).
- **Rate Limiting:** Implemented global rate limiting to the entire API (300 req/min) and strict limits on Auth flows (10 req/15 min) along with existing public feed limits.

### 2. Authentication & Authorization (RBAC)
- **Refresh Token Flow:**
  - Configured short-lived Access Tokens (15 min) and long-lived Refresh Tokens (7 days).
  - Stored Refresh Tokens securely in the database.
  - Added new `/api/v1/auth/refresh` endpoint and updated `/login`, `/register`, and `/logout` to support the new token paradigm.
- **Role-Based Access Control (RBAC):**
  - Updated standard user schema to include a `role` column (defaults to `"user"`, options: `"user"` | `"admin"`).
  - JWT payload now includes `role` field via `TokenService` ÔÇö zero extra DB calls per request.
  - Created `allowRoles(...roles)` middleware (in `middleware/auth.js`): checks `req.user.role`, returns `403 FORBIDDEN` on mismatch.
  - Built dedicated **Admin Route** (`routes/admin.js`) mounted at `/api/v1/admin`, protected by both `auth` + `allowRoles('admin')`:
    - `GET /admin/users` ÔÇö Paginated user list
    - `DELETE /admin/users/:id` ÔÇö Delete user & revoke all their refresh tokens (self-deletion blocked)
    - `PATCH /admin/users/:id/role` ÔÇö Promote/demote any user
    - `DELETE /admin/public/:id` ÔÇö Global post moderation (invalidates feed cache)

### 3. Observability & Documentation
- **Pino Logger:** Dropped `console.log` in favor of a structured, colorized (in-dev) Pino logger for tracing request anomalies.
- **Swagger Documentation:** Actively served OpenApi specifications using Swagger UI at `/api-docs`.
- **Structured Request Logger (`middleware/requestLogger.js`):**
  - Replaced `morgan` with a custom, Pino-backed structured logger.
  - Generates a unique **Request ID** (`crypto.randomUUID()`) per request, attached as `X-Request-Id` response header and `req.id`.
  - Logs **both directions**: `Ôćĺ request IN` (method, url, ip, userAgent, userId) and `ÔćÉ response OUT` (status, route, durationMs).
  - **Slow endpoint detection**: flags any request exceeding `SLOW_THRESHOLD_MS` (default 500ms) with `slow: true` at `WARN` level.
  - **Log level routing**: `info` (normal), `warn` (4xx or slow), `error` (5xx).
  - **Noise suppression**: skips `/health`, `/api-docs`, and `/favicon` routes automatically.
- **Sentry Error Tracking (`utils/sentry.js`):**
  - Integrates `@sentry/node` for automatic unhandled error capture.
  - **Fully optional** ÔÇö silently no-ops when `SENTRY_DSN` is not set (zero impact on dev/test).
  - All unexpected 5xx errors in `errorHandler.js` are reported to Sentry with `requestId`, `method`, `path`, and `userId` context.
  - Ignores known non-bugs: `TokenExpiredError`, `JsonWebTokenError`, `UnauthorizedError`.
- **Environment Variables Added** (`.env.example` created):
  - `LOG_LEVEL`, `LOG_REQUESTS`, `SLOW_THRESHOLD_MS`, `SENTRY_DSN`, `SENTRY_ENVIRONMENT`, `SENTRY_TRACES_RATE`.

### 4. Application Performance
- **Response Compression:** Implemented global payload compression using `compression` middleware inside `app.js`.
- **Redis Caching Layer:** Introduced caching for high-traffic analytical dashboard routes and the public feed to save resources by avoiding repeated database reads (TTL set to 60s).
- **Database Optimizations:** Verified index scripts for highly queried constraints, enabling indices on user associations (`user_id`) and chronological sorting (`created_at`).

### 5. API Versioning & Future-Proofing
- **Folder Structure:** Created `routes/v1/` and `routes/v2/` subdirectory scaffold to support future breaking changes without disrupting existing consumers.
- **Version Middleware (`middleware/apiVersion.js`):**
  - Detects version from URL (`/api/v1/`, `/api/v2/`) or `X-API-Version` request header.
  - Stamps every response with `X-API-Version` header.
  - Injects `X-API-Deprecated: true` and `X-API-Sunset: <date>` headers on routes scheduled for removal (configured via `DEPRECATED_VERSIONS` map).
- **`routes/v1/index.js`:** Thin delegation layer ÔÇö re-mounts all existing flat route files under the versioned router. Zero route logic was rewritten.
- **`routes/v2/index.js`:** Scaffold router that returns clear `501 NOT_IMPLEMENTED` for any unimplemented v2 endpoint, guiding clients back to v1.
- **Backward Compatibility:** All existing `/api/v1/*` flat mounts remain in `app.js` ÔÇö no breaking changes to live consumers.

- **Jest and Supertest Setup:** Altered package.json and `db/connection.js` to actively support a safe, isolated, transparent `:memory:` testing sequence for better-sqlite3 environments.
- **Stable Test Suite:**
